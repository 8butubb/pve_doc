# 局域网架构

## 核心理念

**Cloudflare 泛域名 + Lucky 反向代理 + AdGuard 网络过滤**

## 完整访问链路

```
客户端 (192.168.1.10)
       │
       │ ① 输入 https://dsm.butubb.cn
       ▼
系统 DNS 查询 → 192.168.1.2 (AdGuard)
       │
       ▼
AdGuard Home → 不解析 *.butubb.cn → 透传到 Cloudflare
       │
       ▼
Cloudflare → *.butubb.cn 泛解析 → 192.168.1.2
       │
       ▼
客户端访问 192.168.1.2:443
       │
       ▼
Lucky → SSL 终止（自动证书）→ Host 头匹配 → 反代到 192.168.1.50:5000
       │
       ▼
群晖 NAS 返回数据
```

## DNS 解析链路

```
客户端 DNS 查询
       │
       ▼
① AdGuard Home (192.168.1.2)
   • 仅做网络过滤（广告/恶意域名）
   • 不解析 *.butubb.cn
   • 透传到 Cloudflare
       │
       ▼
② Cloudflare (1.1.1.1)
   • *.butubb.cn 泛解析 → 192.168.1.2
       │
       ▼
③ 客户端收到 192.168.1.2
```

## Cloudflare 配置

```
A      *.butubb.cn         192.168.1.2
CNAME  pve.doc.butubb.cn   username.github.io
```

## AdGuard Home 配置

```yaml
upstream_dns:
  - 1.1.1.1
  - 8.8.8.8
```

**功能**：
- ✓ 广告/恶意域名拦截
- ✗ 不解析内网域名（透传到 Cloudflare）

## Lucky 反代配置

```
规则 1：
  域名：dsm.butubb.cn
  后端：192.168.1.50:5000
  SSL：✓（自动申请）

规则 2：
  域名：router.butubb.cn
  后端：192.168.1.1:80
  SSL：✓
```

## 网络拓扑

```
LAN (192.168.1.0/24)
├── VM-1/2/3 (客户端，DNS=192.168.1.2)
├── "network" LXC (192.168.1.2)
│   ├── Tailscale (100.64.1.2)
│   ├── AdGuard Home (网络过滤)
│   └── Lucky (反代 :443)
├── 群晖 NAS (192.168.1.50:5000)
├── 路由器 (192.168.1.1:80)
└── 绿联 NAS (192.168.1.60, 备份专用)
```

## "network" 容器

| 属性 | 值 |
|------|-----|
| LAN IP | 192.168.1.2 |
| Tailscale IP | 100.64.1.2 |
| 运行服务 | AdGuard + Tailscale + Lucky |

## 端口规划

| 服务 | 端口 | 说明 |
|------|------|------|
| AdGuard DNS | 53 | 网络过滤 |
| AdGuard Web | 3000 | 管理界面 |
| Lucky HTTPS | 443 | 反代入口 |
| Tailscale | 41641/UDP | VPN 通信 |

## 客户端配置

```bash
# /etc/resolv.conf
nameserver 192.168.1.2
```

或在路由器 DHCP 中统一下发 DNS 为 `192.168.1.2`。
