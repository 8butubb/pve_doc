# Tailscale 组网架构

## 核心理念

让**外部客户端**（手机/笔电）通过 Tailscale VPN，使用**与 LAN 相同的域名**访问内部服务。

- **Cloudflare**：`*.butubb.cn` 泛解析到 192.168.1.2
- **AdGuard Home**：组网 DNS 出口，仅做网络过滤
- **Tailscale**：WireGuard 隧道 + 子网路由
- **Lucky**：SSL 终止（自动证书）+ Host 头反代

## 完整访问链路

```
手机 (100.64.1.51)
       │
       │ ① 输入 https://dsm.butubb.cn
       ▼
系统 DNS 查询 → 127.0.0.1:100 (Tailscale)
       │
       ▼
Tailscale MagicDNS → 100.100.100.100
       │
       ▼
AdGuard Home (192.168.1.2) → 不解析 *.butubb.cn → 透传到 Cloudflare
       │
       ▼
Cloudflare → *.butubb.cn → 192.168.1.2
       │
       ▼
客户端访问 192.168.1.2:443
       │  192.168.1.0/24 在子网路由中
       ▼
Tailscale 隧道 → 100.64.1.2 (network)
       │
       ▼
Lucky → SSL 终止（自动证书）→ 反代到 192.168.1.50:5000
       │
       ▼
群晖 NAS 返回数据
```

## DNS 解析链路

```
① 客户端 DNS → 127.0.0.1:100 (Tailscale)
② Tailscale MagicDNS → 100.100.100.100
③ AdGuard Home (192.168.1.2) → 网络过滤 → 透传
④ Cloudflare → *.butubb.cn → 192.168.1.2
⑤ 客户端收到 192.168.1.2
```

## 网络拓扑

```
外部设备
├── 笔记本 (100.64.1.50)
└── 手机 (100.64.1.51)
       │
       │ Tailscale 隧道
       ▼
"network" LXC (100.64.1.2 / 192.168.1.2)
├── Tailscale 守护进程
├── AdGuard Home (网络过滤)
└── Lucky (反代 :443)
       │
       ▼
LAN 后端服务
├── 群晖 NAS (192.168.1.50)
├── 路由器 (192.168.1.1)
└── 其他服务
```

## 配置要点

### Tailscale 节点配置

```bash
tailscale up \
  --advertise-routes=192.168.1.0/24 \
  --accept-dns=true \
  --accept-routes=true

echo 'net.ipv4.ip_forward = 1' >> /etc/sysctl.conf
sysctl -p
```

### Tailscale 管理控制台

1. 批准子网路由 192.168.1.0/24
2. DNS 上游指向 192.168.1.2 (AdGuard)

### Cloudflare 配置

```
A      *.butubb.cn         192.168.1.2
CNAME  pve.doc.butubb.cn   username.github.io
```

### AdGuard Home 配置

```yaml
upstream_dns:
  - 1.1.1.1
  - 8.8.8.8
```

### Lucky 反代配置

```
规则 1：
  域名：dsm.butubb.cn
  后端：192.168.1.50:5000
  SSL：✓（自动申请）
```

## 安全特性

- 端到端加密（WireGuard）
- 设备认证 + ACL 控制
- DNS 透传到 Cloudflare
- 网络过滤（AdGuard）

## 故障排查

```bash
# 测试 DNS 解析
nslookup dsm.butubb.cn
# 应返回 192.168.1.2

# 测试到 network 节点连通性
tailscale ping 100.64.1.2

# 检查子网路由
tailscale status | grep subnet

# 检查转发
cat /proc/sys/net/ipv4/ip_forward
# 应为 1
```
