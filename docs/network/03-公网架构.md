# 公网访问策略

## 核心理念

**零公网暴露 + Cloudflare 泛域名**：

- 内网服务域名 `*.butubb.cn` 解析到私网 IP 192.168.1.2（公网不可达）
- 静态内容 `pve.doc.butubb.cn` 指向 GitHub Pages
- 必须通过 Tailscale VPN 接入才能访问内网服务

## 访问模型

```
公网层
├── GitHub Pages (pve.doc.butubb.cn) → 公开访问 ✓
└── 内网服务公网入口 → 无 ✗

唯一通道：Tailscale VPN
       │
       ▼
"network" LXC
├── Tailscale 节点
├── AdGuard (网络过滤)
└── Lucky (反代 :443，自动证书)
       │
       ▼
内部服务层
├── 群晖 NAS
├── 路由器
└── 其他服务
```

## 三种访问场景

### 1. 公开内容（GitHub Pages）

```
用户浏览器
    │ HTTPS
    ▼
Cloudflare CDN → GitHub Pages
```

### 2. 内网服务 - 未连 VPN（拒绝）

```
手机（未连 VPN）
    │ 输入 dsm.butubb.cn
    ▼
Cloudflare → *.butubb.cn → 192.168.1.2（私网 IP）
    │
    ▼
私网 IP 公网不可达 → 访问失败 ✗
```

### 3. 内网服务 - 已连 VPN（成功）

```
手机（已连 VPN）
    │ 输入 dsm.butubb.cn
    ▼
Tailscale DNS → AdGuard → Cloudflare → 192.168.1.2
    │
    ▼
Tailscale 隧道 → network 容器 → Lucky 反代 → 后端服务
    │
    ▼
访问成功 ✓
```

## 核心原则

| 原则 | 说明 |
|------|------|
| 零公网暴露 | 无端口转发、无 DMZ |
| 私网 IP 解析 | 域名解析到 192.168.1.2（公网不可达） |
| VPN 强制接入 | 必须 Tailscale 认证 |
| 静态内容分离 | 文档走 GitHub Pages |

## Cloudflare 配置

```
A      *.butubb.cn         192.168.1.2  (私网 IP，公网不可达)
CNAME  pve.doc.butubb.cn   username.github.io
```

⚠ 路由器上**不要**配置端口转发，否则破坏零公网暴露。

## 安全屏障

| 屏障 | 说明 |
|------|------|
| ① 路由器 | 无端口转发、无 DMZ |
| ② DNS | 解析到私网 IP（公网不可路由） |
| ③ VPN | Tailscale 认证 + ACL |

## 访问控制

| 用户 | 文档博客 | 内网服务 |
|------|----------|----------|
| 匿名 | ✓ | ✗ 不可达 |
| VPN 成员 | ✓ | ✓ |
| 管理员 | ✓ | ✓ 完全访问 |

## 故障排查

```bash
# 检查 DNS 解析
dig dsm.butubb.cn @1.1.1.1
# 应返回 192.168.1.2

# 检查 VPN 状态
tailscale status
tailscale ping 100.64.1.2

# 检查 service
systemctl status lucky
ss -tlnp | grep 443
```
